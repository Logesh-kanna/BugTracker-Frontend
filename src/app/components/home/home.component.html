ABOUT

  • This is a Microservices-based application built using Angular, Java Spring Boot, PostgreSQL, JWT, Cloudinary, SMTP, Eureka, and OpenFeign. The application manages an Admin Panel and a Bug Tracker system, each with distinct user roles and functionalities.

Admin Panel

  • The Admin Panel is used to manage client requests and assign them to the system. It features two types of users:
  • Admin: Responsible for reviewing and accepting client requests.
  • User: A role that can view client information, which is stored in separate databases for each client.

The process works as follows:

1. User accesses and reviews client information.

2. The Admin reviews the information and, if everything is satisfactory, accepts the request.

Bug Tracker

  • The Bug Tracker is designed for managing projects, bugs, and user assignments, and includes three distinct user roles:
  • Admin: Responsible only for creating and deleting users in the system.
  • Manager: Responsible for handling project-related activities, bug creation, and management.
  • User: Works on assigned projects and resolves specific bugs.

Key Features:

1. Admin Role:

  • The Admin in the Bug Tracker has limited access. They are only responsible for creating and deleting users within the system.

2. Project Creation and User Assignment:

  • Manager users can create new projects, assign users to projects, and manage bugs.

3. Bug Management:

Bugs can be created with specific details, including:

  • Status
  • Priority
  • Due Date (automatically assigned based on priority)
  • Attachments (images and videos)
  • Assignment to users working on the project.

4. Email Notifications:

  • Customized email IDs and passwords are sent to users when they are assigned to a project or given access to the system.
